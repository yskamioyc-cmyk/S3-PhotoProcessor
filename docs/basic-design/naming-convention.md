## 命名規則の目的について
- 本プロジェクトにおける命名規則の設定は、業務や仕様の分類・区分に合わせた記号の選定を一括して行うことで、以降の設計書や要件定義書の可読性を向上させることを目指す。
- また、すでに作成済みの自作環境をテンプレートとして使用するため、順序が逆転するが、一部のIDなどの命名規則については、すでに作成済みの環境構築テンプレートの規則に倣うものとする。
- それらに該当しない、業務管理、機能要件、非機能要件等に関する命名規則については本ドキュメント内で新規に設定し、以降のドキュメントは本ドキュメントを参照するものとする。

## システム名およびサブシステム名について
当該システムのシステム名称、サブシステム名称は以下の通り定義します。

### システム名
| システム名称 | 説明 |
| :-- | :-- |
| S3-PhotoProcessor | 端末上に存在する画像データを、AWSを想定したバケットに格納し、サーバー上での画像加工機能とともにユーザーに提供する一連のシステム。 |

### サブシステム名
| システムID | 名称 | 説明 |
| :-- | :-- | :-- |
| AUTH | 認証サブシステム | ユーザー登録、ログイン、ログアウト、パスワードリセット |
| IMG | 画像管理サブシステム | 画像アップロード、一覧表示、削除、メタデータ編集 |
| PROC | 画像加工サブシステム | リサイズ処理、フィルタ適用、加工パラメータ管理 |

| IMG-010 | 画像アップロード | 端末から画像をアップロードする。 |
